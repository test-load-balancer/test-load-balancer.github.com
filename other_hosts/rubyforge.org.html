tlb.rb - 'Test Load Balancer' for Ruby

tlb.rb is the TLB Ruby support library. It allows load-balancing of Test::Unit, RSpec-1.x or RSpec-2.x test suites among multiple hosts that share the test-running load.

TLB makes multiple partitions of the tests to-be-run by the subject test-task, and allows execution of each such partition on a different machine/process. Since each such partition executes only a subset of tests, the time each one takes is 'total-serial-execution-time / number-of-partitions'.

All such partitions execute parallely and finish at almost the same time, hence dividing the build time by number-of-partitions made.

So assuming serial-execution of the entire test-suite for a project 'foo' takes 10 miutes, and there are 5 machines available to load-balance these tests, TLB can have the build finish in just 2 minutes by insuring:

  1. each machine/process executes only one subset of tests
  2. no tests are skipped
  3. no tests are executed on more than one machine

It can be used on the CI server(to run tests parallely, hence cutting build time) or can be used in other innovative ways, like with a Capistrano recipe that invokes test-task on multiple machines at the same time, so that TLB can engage to have each machine run only a subset of tests, and provide developers quicker feedback on local changes.

TLB Ruby support library is distributed through RubyGems.org as multiple framework-specific gems:

  • tlb-rspec2: Support library for RSpec-2.x(2.4, 2.3 etc)
  • tlb-rspec1: Support library for RSpec-1.x(1.3.1, 1.3.0 etc)
  • tlb-testunit: Support library for Test::Unit
